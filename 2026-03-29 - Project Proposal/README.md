This project is a browser-based Chess Game with AI designed to provide users with a complete and interactive chess experience while also demonstrating practical full-stack software development skills. The main objective of the project is to build a playable chess application where users can log in, start a new game or resume a saved game, choose their preferred game settings, and compete against an AI opponent with multiple difficulty levels. The project combines user interface design, game rule management, backend development, AI decision-making, and cloud database integration into a single system.
The application is designed to ensure that the game of chess is both functional and user-friendly. The user will have the ability to choose various key aspects of the game before commencing play. This will include the time mode of play, the type of chessboard theme to be used, the color of the pieces for the player's side of the board, and the difficulty level of the computer's AI. The game supports multiple timer styles such as Blitz, Rapid, and Classical, allowing the user to experience different competitive formats. It also includes pause functionality, undo and redo options, sound effects, move history, captured-piece tracking, and visible status messages for events such as check, checkmate, and timeout. These features are designed to improve gameplay clarity and create an experience closer to real online chess platforms. In addition, once a timed game ends, the user is given the option to extend the clock and continue playing.
Another important goal of this system is to have persistence and user continuity. The application has features like login and registration so that each player has his or her unique account. The user's details and the saved game state are stored in MongoDB Atlas. This allows the project to use a real cloud database rather than temporary memory storage. This enables a player to exit the game, log out, and log in again to continue the game from the point he or she left. The saved state includes the position of the board, move history, timers, selected side, theme, difficulty level, pause state, and other game state details. This feature of the project makes it more realistic.
From an architectural point of view, the system uses a client-server model. In this system, the frontend is built with HTML, CSS, and JavaScript. It is responsible for rendering the chessboard, handling user interactions, and making API calls to the backend. The backend is built with Node.js and Express.js. It has REST API endpoints for authentication, AI moves, and saved games. It uses MongoDB Atlas for the database layer to store user accounts and game sessions. It uses chess.js for chess logic validation, generating legal moves, and handling chess game states. It uses the minimax algorithm with alpha-beta pruning for the AI module. Difficulty level is achieved by modifying search depth, evaluation strength, move ordering, and controlled randomness for AI moves. It allows the AI to behave differently for easy, normal, and hard difficulty levels.
